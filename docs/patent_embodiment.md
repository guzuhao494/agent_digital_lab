# 智能体数字实验室岩爆预警方法实施例

## 一、系统组成

本实施例提供一种面向 TBM 隧道施工岩爆风险的智能体数字实验室系统，系统包括多源输入层、统一状态构建层、多智能体协同层、数字实验室核心层、闭环输出层和反馈校正层。

多源输入层用于获取微震监测数据、TBM 掘进参数数据和地质结构数据。微震监测数据包括事件时间、空间坐标、释放能、震级和破裂机制类型；TBM 掘进参数数据包括掌子面里程、推进速度、刀盘转速、推力、扭矩、贯入度和支护压力；地质结构数据采用 JSON 形式描述当前区段岩性、地应力、断层、蚀变带、结构面、构造影响半径及机理图谱节点。

统一状态构建层用于对多源数据进行时空对齐、特征提取和统一状态编码，形成当前围岩岩爆孕育状态。该状态以 `state_snapshot` 表示，包含当前时间戳、掌子面里程、微震特征、TBM 工况特征、地质特征和风险上下文。

多智能体协同层包括微震感知智能体、掘进工况智能体、地质认知智能体、机理匹配智能体、推演实验智能体、预警决策智能体和反馈校正智能体。各智能体通过 OpenClaw 运行时统一调用，具体规则、统计和推演逻辑在后端实现。

数字实验室核心层用于基于候选机理路径构建多个反事实推演场景，模拟不同施工参数、支护参数和地质扰动条件下未来岩爆风险变化。

闭环输出层用于综合各场景风险等级、风险区段、主导机理和处置收益，生成最终预警结果和处置方案。

反馈校正层用于在获取后续真实监测结果后，对功能智能体参数、机理路径权重和推演场景评价结果进行更新，形成闭环优化。

## 二、统一状态构建方法

系统首先读取微震、TBM 和地质三类数据。以 TBM 最新记录的时间和掌子面里程作为当前状态基准，截取邻近时间窗内的微震事件和近期 TBM 参数序列，并确定当前里程影响范围内的地质构造体。

微震状态特征包括时间窗内事件数、累计释放能、最大释放能、空间重心、掌子面前后事件比例、左/右/拱顶/拱底分布比例，以及剪切、拉伸、混合破裂机制占比。

TBM 状态特征包括推进速度、刀盘转速、推力、扭矩、贯入度、各参数波动幅度、异常分数和扰动指数。

地质状态特征包括当前里程对应地质体类型、是否靠近断层、是否靠近蚀变带、是否靠近结构面、结构面产状编码和地质复杂度评分。

上述特征组合成统一状态对象：

```json
{
  "timestamp": "...",
  "chainage": 36240.0,
  "microseismic_features": {},
  "tbm_features": {},
  "geology_features": {},
  "risk_context": {}
}
```

该统一状态对象作为后续所有智能体的共同输入。

## 三、多智能体协同方法

微震感知智能体根据时间窗内微震数据输出微震活跃度、聚集区坐标、高能事件摘要、破裂机制统计和初步风险分。

掘进工况智能体根据 TBM 时序参数输出当前工况标签、扰动强度、异常掘进波动判断，以及 TBM 扰动与微震风险之间的耦合提示。

地质认知智能体根据地质 JSON 和当前里程输出当前附近地质体摘要、特殊结构风险标签，以及对岩爆风险的增强或削弱因子。

机理匹配智能体根据统一状态对象和机理规则库计算多个候选机理得分，输出候选岩爆孕育机理路径、主导机理和触发证据。第一版采用机理规则库和相似度打分，不依赖图数据库；后续可迁移到机理图谱。

## 四、数字实验推演方法

推演实验智能体基于候选机理路径和当前状态构建至少五类反事实推演场景：

- 场景 A：维持当前掘进参数。
- 场景 B：降低推进速度 10%。
- 场景 C：降低推进速度并加强局部支护。
- 场景 D：假设前方 10m 存在不利结构面。
- 场景 E：假设地质扰动减弱。

对每个场景，系统输出未来 1h、3h 和 6h 的风险评分、高风险区间、预期主导机理和建议操作，并计算该场景的峰值风险作为综合评价指标。

## 五、闭环预警与反馈更新

预警决策智能体汇总微震感知、工况识别、地质认知、机理匹配和数字实验分支结果，输出最终风险等级、风险位置、风险机理、推荐方案、方案置信度和补充监测建议。

反馈校正智能体在获取后续真实监测结果后，对微震能量权重、TBM 扰动权重、机理路径权重和推演场景评价结果进行调整。该过程使系统从一次性预测变为闭环优化方法，适合在专利中表述为“数字实验室驱动的智能体协同岩爆预警与施工控制优化方法”。

## 六、当前 MVP 对应关系

- 多源输入：前端支持上传微震 CSV、TBM CSV 和地质 JSON；未上传时使用样例数据。
- 统一状态构建：后端生成 `state_snapshot` 并作为所有 agent 输入。
- 智能体协同：所有 agent 通过 OpenClaw 运行时边界执行，业务逻辑在后端实现。
- 数字实验：生成五类反事实场景，并输出 1h、3h、6h 风险。
- 闭环输出：输出风险等级、风险区段、主导机理、推荐处置方案、置信度和补采建议。
- 空间展示：前端用 Three.js 展示隧道、微震点云、地质体和风险热区。
